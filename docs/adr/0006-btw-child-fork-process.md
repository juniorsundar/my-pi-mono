# BTW uses a child `pi --fork` process instead of an in-process agent session

BTW spawns an isolated child `pi` process via `--fork` to resolve side-questions. The child gets the full conversation history (via session fork), runs with `--exclude-tools edit,write` for safety, and streams JSON events back to the parent extension. The alternative — running an in-process `AgentSession` — would avoid spawn overhead and give tighter integration, but it would share the parent's tool registry (making read-only enforcement fragile), risk resource contention with the main agent loop, and make async lifecycle management significantly harder. The child process gives clean isolation: separate memory, separate tools, separate lifecycle. The spawn cost (~1s startup) is acceptable for a side-question that typically runs for seconds to minutes.
